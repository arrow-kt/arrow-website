# Arrow 2.2.2

Arrow 2.2.2 is out!
The main focus of this release is bugfixing — especially around the Arrow Optics plugin —
but also comes with a few goodies.

**Integration with [Ktor type-safe routing](https://ktor.io/docs/server-resources.html).**
The new [`arrow-raise-ktor-server-resources` module](https://apidocs.arrow-kt.io/arrow-raise-ktor-server-resources/index.html)
provides integration between Arrow's Raise mechanism and Ktor's definition of paths as resources.

**More platforms.** Arrow is now published for all platforms
[supported by Kotlin/Native](https://kotlinlang.org/docs/native-target-support.html#target-tiers).

**More context parameters.**
- The [`arrow.core.raise.context`](https://apidocs.arrow-kt.io/arrow-core/arrow.core.raise.context/index.html) package
  now exposes the same API as the [`arrow.core.raise` package](https://apidocs.arrow-kt.io/arrow-core/arrow.core.raise/index.html),
  but using context parameters. Up to this version some functions were missing.
- The new [`arrow.fx.coroutines.resource.context`](https://apidocs.arrow-kt.io/arrow-fx-coroutines/arrow.fx.coroutines.resource.context/index.html)
  package exposes the [resource management](https://arrow-kt.io/learn/coroutines/resource-safety/) capabilities of
  Arrow using context parameters.

**Deprecations around lists.**
- There has been some ongoing problems with [`unweave`](https://apidocs.arrow-kt.io/arrow-core/arrow.core/unweave.html),
  because of its unclear semantics. Since these semantics haven't been clarified for a long time, 
  we have decided to deprecate it.
- The [`foldLeft`](https://apidocs.arrow-kt.io/arrow-core/arrow.core/-non-empty-list/fold-left.html) function
  for `NonEmptyList` has been deprecated in favor of
  [`foldLeft2`](https://apidocs.arrow-kt.io/arrow-core/arrow.core/-non-empty-list/fold-left2.html).
  This new function has a more general type, since it knows that the reducer will be called at least once.
  In principle, every use site of `foldLeft` could be directly replaced with `foldLeft2`, except those cases in
  which explicit type arguments are used.

Thanks to everybody who has contributed to this release!
